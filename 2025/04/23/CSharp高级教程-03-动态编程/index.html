C# 高级教程 - 03 动态编程

发表于 2025-04-23 分类于 教程 ， C# Waline：
C# 在保持强类型静态语言优势的同时，通过 dynamic 关键字引入了动态编程能力。动态编程在特定场景下极具价值，如与动态语言（Python、JavaScript）互操作、处理 JSON/XML 等动态结构、以及绕过静态类型检查实现灵活调用。

C# 在保持强类型静态语言优势的同时，通过 dynamic 关键字引入了动态编程能力。动态编程在特定场景下极具价值，如与动态语言（Python、JavaScript）互操作、处理 JSON/XML 等动态结构、以及绕过静态类型检查实现灵活调用。本文将深入探讨 C# 动态编程的核心机制、DLR 架构及最佳实践。

dynamic 类型入门

dynamic 在编译期绕过类型检查，所有成员解析被推迟到运行时。这消除了编译时错误，但可能产生运行时异常。

DLR 架构

Dynamic Language Runtime（DLR）是 .NET 中为动态语言提供支持的运行时层。

CallSite<T> 缓存调用绑定结果，让相同操作的后续调用性能接近静态调用。

ExpandoObject —— 动态扩展对象

应用场景：当需要在运行时动态构建对象（如 Web API 返回动态 DTO）时，ExpandoObject 比 Dictionary<string, object> 更自然。

DynamicObject —— 自定义动态行为

通过继承 DynamicObject，可以控制成员调用、属性读写、索引操作等全部动态行为。

IDynamicMetaObjectProvider —— 进阶自定义

直接实现此接口可完全控制元数据绑定流程，适用于构建自定义语言的 .NET 实现、ORM 动态查询构造器等高级场景。

完整实现较为复杂，在绝大多数场景中使用 DynamicObject 已经足够。

性能考量

建议：

  • 不在热点路径（循环/高频调用）中使用 dynamic
  • 使用 dynamic 的场景确保类型在运行时一定匹配
  • 结合 using static 和缓存模式减少动态调用开销

何时使用 dynamic

场景 推荐度 说明
COM 互操作 ✅ 强烈推荐 避免 PIA，代码简洁
与动态语言交互 ✅ 强烈推荐 Python/IronPython 集成
JSON 数据探索 ⚠️ 慎用 快速原型可以，生产推荐强类型
泛型类型参数 ❌ 不推荐 应使用 T + 约束替代
API 返回可变结构 ⚠️ 部分场景 考虑 JsonNode 作为替代

总结：dynamic 为 C# 提供了静态类型世界中的动态弹性。DLR 的缓存机制让性能损失可控，DynamicObject 和 ExpandoObject 则提供了不同层次的自定义动态行为。合理使用动态编程可以大幅简化某些场景的代码量，但应当谨慎评估静态类型替代方案的可行性。

钓鱼日记[23.08.06]

发表于 2023-08-06 更新于 2024-08-26 分类于 钓鱼日记 阅读次数： Valine：

周六：

又到了周末，早早地和表弟约好周六一起去邛崃固驿河边的沙坑野钓。这个沙坑是过去用于采沙的地方，和河道相通，听说已经很多年没有采沙了。沙坑最深处可能达到了十多米。虽然水面不大，但每天都有很多钓友来这里钓鱼。为了尽早占领钓位，我们7点左右就出发了。对于我来说，这是第一次去这个地方，所以心里有些期待。

通过固驿大桥后，我们沿着河边路一直往上游走。尽管才7点，沿河的钓友已经坐满了。整个河边区域都非常适合垂钓，水流很平缓，岸边的树木在阳光下显得特别美丽。河边还有一条钓友们踩出来的泥路，顺着河流蜿蜒向上，我觉得肯定有很多鱼躲藏在那里。虽然没能拍照，但下次一定要找个时间来这个河边探钓一下。

大约走了2公里，我们抵达了今天的目的地——沙坑。从外面看，沙坑有个铁门，之前都是关着的，但最近开放了，可以直接开车去水边，非常方便。

沙坑一边靠着山，靠山的那一边应该是最深的，不适合钓鱼，而且也比较危险。

靠近河边的一边，已经坐满了很多钓友，看起来应该比较容易钓到鱼。

我们选了一个侧面入口处的钓位，感觉这个位置也不错，通过找底发现前面3米处是一个高坎，上面2米多深的水位，坎下将近有5米，我用13.5尺的狮子吼刚好钓到坎边，总体来说今天这个沙坑的鱼情并不好，周围钓友都说口比较少，我们也一样，可能要等好一会才会来一口，而且也经常钓不上来鱼，但是这个沙坑的鱼种确实很丰富。收竿的时候，我钓到了好几个品种的鱼，包括鲫鱼、鲤鱼、趴地虎、白条、黄尾鲴、土凤、三角锋、鳑皮、河虾和螃蟹。我的表弟甚至钓到了一条翘嘴。还有几条小黑鱼始终在我们钓位前面的水草下晃荡。我们钓上来的这些鱼个体都不大，大点的二，三两，但都是野生河鱼还是很巴适的，中途跑了四条鱼，我感觉要大一些，特别是最后跑的那条，起初又是挂底的感觉，抖动了两下鱼竿才开始游动，鱼没发力感觉不到大小，弯着鱼竿感觉到鱼在水下慢慢游动，持续几秒就脱钩了。唯一不好的地方是河边没有遮阳的地方，尽管打了伞，太阳还是很大，很热。也许这是今天鱼口不好的原因之一吧。

- C# 多态性

多态是同一个行为具有多个不同表现形式或形态的能力。

多态性意味着有多重形式。在面向对象编程范式中，多态性往往表现为”一个接口，多个功能”。

多态性可以是静态的或动态的。在静态多态性中，函数的响应是在编译时发生的。在动态多态性中，函数的响应是在运行时发生的。

在 C# 中，每个类型都是多态的，因为包括用户定义类型在内的所有类型都继承自 Object。

    同一个事件发生在不同的对象上会产生不同的结果。

- 静态多态性

在编译时，函数和对象的连接机制被称为早期绑定，也被称为静态绑定。C# 提供了两种技术来实现静态多态性。分别为：

  • 函数重载

  • 运算符重载

钓鱼日记[23.07.22]

发表于 2023-07-22 更新于 2024-08-26 分类于 钓鱼日记 阅读次数： Valine：

又到了周末，虽然天气炎热，但钓鱼人的热情并没有被打消。上个周末去钓大鲤鱼的计划没能实现，我总结了一周，觉得主要原因是没有打窝。所以今天特意带了红虫颗粒和谷麦窝，准备在新的钓位上再次尝试。

9点到达水库边，太阳早早地出来了。为了防晒，我选择了一个背靠大树的钓位，面对着水库。钓位的左侧是一个小湾子，形状像一个字母“C”，岸边都是树。本来这个湾子是个很好的钓位，但水质不好，垃圾也很多，连浮漂都站不起来。所以我选了一个稍微往外一点的位置。不多废话，选好钓位后，我先打窝。在3.6米的位置，我打了几把红虫颗粒，准备钓小鲫鱼。在6.3米的位置，我用长竿挂上玉米，准备钓大鲤鱼。

一番准备之后，已经是9点半了。我决定先用12尺的玄武人Y钓一些小鱼回去喂乌龟。撮了两个大饵，抛入水中先找底。结果浮漂一入水就全没入水中，一提竿大弯弓水下没动静，感觉像挂底，习惯性的拿着鱼竿向上轻弹两下，突然一股具大的力道从鱼竿传到手撑，鱼竿差点脱手而去，鱼线带着嗖嗖嗖的声音往外串，，我急忙调整鱼竿的角度，左右晃动鱼竿，希望能让鱼竿弯起来。但感觉就像是牵了一头不听指挥的牛，一直往前冲。结果鱼脱钩了，4号袖钩都被拉直了。看着水面上翻起的夹着浑水的大水花，我的内心久久不能平静。过了几分钟，等我心情平复了一些，我分析了一下原因。这其实是跑鱼的正常现象。12尺的玄武人Y有点短，主线0.8号，子线0.4号，加上4号袖钩，这样的配置确实只适合钓小鱼。再加上水深只有60厘米，很难有鱼游动的空间。一想到这些，我不再那么懊恼了。钓鱼玩的就是一个过程，于是我收拾好心情，继续钓。

刚刚跑了鱼，窝里肯定已经没有鱼了。于是我准备去玩路亚。上次的路亚装备还在，我准备玩飞蝇。出人意料的是，路亚鱼口很好，还解锁了鲈鱼。

- C# 继承

继承是面向对象程序设计中最重要的概念之一。继承允许我们根据一个类来定义另一个类，这使得创建和维护应用程序变得更容易。同时也有利于重用代码和节省开发时间。

当创建一个类时，程序员不需要完全重新编写新的数据成员和成员函数，只需要设计一个新的类，继承了已有的类的成员即可。这个已有的类被称为的基类，这个新的类被称为派生类。

继承的思想实现了 属于（IS-A） 关系。例如，哺乳动物 属于（IS-A） 动物，狗 属于（IS-A） 哺乳动物，因此狗 属于（IS-A） 动物。

- 基类和派生类

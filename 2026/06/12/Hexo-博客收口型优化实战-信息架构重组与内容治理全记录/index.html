Hexo 博客收口型优化实战：信息架构重组与内容治理全记录

发表于 2026-06-12 分类于 Hexo博客搭建 Waline：
ByteFisher 博客上线2年后迎来"收口型"优化——不追求新功能，而是对信息架构、SEO 质量、废弃资产和基础设施做系统性清理。记录首页改为内容入口矩阵、14+ 篇文章 description 批量重写、旧图床链接清零、tools 目录瘦身等全部过程。

前言

ByteFisher 博客从最初的 Hexo 基础静态站一路迭代，到现在已经覆盖了技术教程、钓鱼日记、钓鱼相册、鱼获 Dashboard、钓点地图、抖音视频、AI 小游戏、AI 问答助手、热门文章、PWA 等多个功能模块。文章总量突破 150+，线下工具脚本接近 20 个，第三方服务依赖 6 个以上。

功能越来越多，但问题也随之暴露：

  • 首页还是时间流——150 篇文章按时间排列，新访客不知道从哪开始看。
  • 文章摘要参差不齐——有的空着，有的截断，有的塞了正文片段。
  • 旧图床链接零星残留——以前用 iili.io，切到 img.bytefisher.top 后还有漏网之鱼。
  • tools 目录积灰——脚本越来越多，三分之一已经不再使用。
  • 基础设施欠账——PJAX cacheBust 没开、PWA 缓存策略过时、CI/CD timezone 没设。

本轮优化的核心理念是”收口”：不追加新功能，对已上线的内容、配置、代码做系统性清理。本文记录这次优化的完整过程。

一、信息架构重组：从时间流到入口矩阵

1.1 背景：一个首页只有时间线的博客

优化前的 index.swig 结构很简单：

Hero 区域放了几个跳转按钮和技术统计，下面就是按日期排列的文章列表。对于知道自己在找什么的老读者没问题，但新访客面对 150 篇文章的时间流，往往不知所措。

1.2 设计思路：按入口引导 + 专题推荐

参考一些内容型网站的布局方式，把首页从上到下拆成四个区域：

  1. Hero 品牌区——保留头像、打字动画和全站数据统计
  2. 内容入口 Grid——教程、钓鱼、游戏、视频四张卡片，按主题分流
  3. 精选专题——游戏开发、C# 基础、Hexo 实战、AI 编程四个系列推荐
  4. 最新文章 + Featured 推荐

关键是第二和第三区——它们让新访客在 1 秒内理解”这个博客有什么”，并且两次点击就能进入任意核心系列。

二、SEO 与内容治理：14+ 篇文章摘要重写

2.1 问题：description 质量参差不齐

经过前几轮优化，文章 description 的覆盖率已经很高（95% 以上），但质量不行。

用 rg -n "^description:" source/_posts -g "*.md" 逐一检查，发现几类典型问题：

问题类型 示例 影响
空描述 description:（仅字段名，无内容） 搜索引擎用正文前 160 字，可能截断不完整
截断残片 以半句话结尾，有明显断句痕迹 搜索结果摘要读不通
正文污染 混入图片描述、代码片段、链接 URL 降低搜索引擎对页面主题的判断
过长 超过 160 字，搜索引擎也会截断 关键信息可能被截掉

四、基础设施加固

4.1 PJAX CacheBust：修复文章消失 Bug

线上发现一个诡异问题：新发布的文章”ByteFisher AI 编程实战（二）”首次加载正常显示，切换到其他页面再回主页就消失了，刷新后又出现。

追到 themes/next/layout/_scripts/pjax.swig 发现 cacheBust: false。PJAX 的 XHR 请求不会加时间戳参数，浏览器直接返回了 HTTP 缓存中的旧首页 HTML——那里面没有新文章。

六、总结

这次收口型优化前后涉及了 30+ 个文件的修改，横跨信息架构、SEO、废弃资产、基础设施、工具链五个维度。几个核心数据：

清理 16 个无用脚本（800+ 行代码）、22 条旧图床链接、1 个废弃页面
修复 PJAX cacheBust、WeChat 40007、Service Worker 版本号

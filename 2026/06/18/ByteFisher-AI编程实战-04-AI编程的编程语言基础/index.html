7.1 跨平台开发注意事项

在 AI 编程中，跨平台差异是常见的踩坑点：

  1. 文件路径：AI 生成的代码中硬编码路径分隔符可能导致跨平台出错。使用 os.path.join() 或 pathlib
  2. 换行符：Windows 用 \r\n，macOS/Linux 用 \n。Git 配置 core.autocrlf 处理
  3. 权限：脚本文件在 macOS/Linux 需 chmod +x 赋予执行权限
  4. 编码：Windows 默认 GBK，AI 生成的代码如果含中文可能乱码。Python 文件头加 # -*- coding: utf-8 -*-

八、常见问题

问题 原因 解决
Python 命令找不到 未添加到 PATH 重新安装，勾选”Add to PATH”或手动添加
pip 安装失败 网络问题 使用镜像源：pip install -i https://pypi.tuna.tsinghua.edu.cn/simple
Git 中文乱码 编码设置 git config --global core.quotepath false
VS Code 终端不识别命令 PATH 未刷新 重启 VS Code 或手动刷新终端

十、AI 编程环境搭建的核心原则

回头看这一篇的内容，总结三个核心原则：

  1. 一次搭建，长期受益：环境搭建是最枯燥但回报最高的一次性投入
  2. 版本管理是第一道防线：AI 生成的代码不可预测，Git 是你最重要的安全网
  3. 用 AI 搭建 AI 环境：遇到安装问题，直接复制报错信息给 AI 工具寻求帮助——这正是本系列要教会你的能力

本章小结

  • Python 是 AI 生态的核心语言，务必掌握虚拟环境管理
  • Node.js 是前端开发和 CLI 工具的基础环境
  • Git 在 AI 编程中比传统开发更重要——每次 AI 操作后 commit，方便 diff 和回滚
  • VS Code 搭配 AI 插件是当前最主流的工作环境
  • 环境搭建是一次性投入，后续所有教程都依赖这套环境
